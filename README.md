# Orbit Watcher

Серверное приложение для мониторинга орбит (в первую очередь GNSS-спутников): регулярно скачивает эфемериды (OMM) с CelesTrak, вычисляет текущее положение спутников и стримит их координаты клиентам в реальном времени через SignalR. Клиентская часть (Blazor) отображает спутники на плоской карте.

> Статус: в разработке (ветка `develop`). Сейчас реализуется минимально полезная версия (MVP).

---

## Что делает проект (идея)

### MVP (минимальная версия)
1. Сервер периодически скачивает OMM-данные спутников с CelesTrak (как минимум наборы GNSS, ~200 спутников).
2. Сервер хранит актуальный снимок (snapshot) орбитальных данных в памяти.
3. Сервер **каждую секунду** вычисляет текущее положение каждого спутника (lat/lon, опционально altitude) на момент `UTC now`.
4. Сервер стримит рассчитанные положения всем подключённым клиентам через **SignalR**.
5. Blazor-клиент подписывается на стрим и отображает маркеры спутников на карте (2D).

### Ближайшие планы (после MVP)
1. **Траектория (ground track)** для выбранного спутника: полвитка назад и полвитка вперёд (визуализация линии на карте).
2. **Инфо по спутнику при наведении/клике**:
   - имя
   - текущее положение
   - скорость (и/или другие derived-параметры)

## Архитектура на высоком уровне

### Сервер (ASP.NET Core)
- **OMM Downloader** (BackgroundService): раз в `ExecuteInterval` скачивает OMM JSON с CelesTrak и обновляет in-memory snapshot.
- **Position Calculator**: используя snapshot + текущее время, вычисляет координаты спутников (SGP4/SDP4).
- **SignalR Hub**: стримит рассчитанные координаты клиентам раз в секунду.

### Клиент (Blazor)
- Подключается к SignalR hub.
- Получает каждосекундные обновления и обновляет маркеры спутников на карте.
- (План) по событию hover/click показывает tooltip/popup с деталями.

## Данные и вычисления

- Источник орбитальных данных: **CelesTrak**, формат **OMM**.
- На основе OMM вычисляется положение спутников во времени (lat/lon[/alt]).
- Важно: OMM — это не “готовые координаты”, а орбитальные элементы + эпоха, поэтому координаты вычисляются на сервере.

---

## Поток данных (runtime)

1. `OmmDownloaderHostedService`:
   - скачивает OMM по списку URL (base + relative uris)
   - парсит OMM
   - формирует словарь спутников по `NORAD Catalog ID`
   - заменяет snapshot целиком (атомарно), чтобы читатели всегда видели согласованное состояние

2. `SatelliteStreamerHostedService`:
   - раз в 1 секунду берёт snapshot спутников
   - считает positions на `DateTime.UtcNow`
   - отправляет пакет координат в SignalR (broadcast всем клиентам)

---

## Формат сообщений

Сервер отправляет событие `satellitePositions` (hub: `/hubs/satellites`), payload:

```json
[
  {
    "noradCatId": 12345,
    "name": "GPS BIIR-2  (PRN 13)",
    "timestampUtc": "2026-05-28T12:34:56Z",
    "lat": 55.75,
    "lon": 37.62,
    "altKm": 20200.0
  }
]
```

## Конфигурация

Используются секции конфигурации (appsettings.*):

- `OmmLoading:ExecuteInterval` — период скачивания OMM (например, раз в 10–60 минут).
- `SatelliteStreaming:ExecuteInterval` — период стриминга в SignalR (по умолчанию `00:00:01`).
- `Celestrack:BaseUri` — базовый URL CelesTrak.
- `Celestrack:RelativeUris` — список путей (наборы данных), которые нужно скачивать (в MVP — GNSS).

> Примечание: конкретные URL/наборы GNSS будут зафиксированы в конфиге, чтобы можно было легко менять источник и состав групп.

## Нагрузочные ожидания (ориентиры)

- Стрим: ~200 спутников × 1 раз/сек.
- Это достаточно небольшой объём для старта, но всё равно важно:
  - не пересылать лишние данные,
  - держать вычисления и сериализацию эффективными,
  - логировать и обрабатывать ошибки так, чтобы стрим не “падал” полностью.

## Запуск сервера и тестового SignalR клиента

1. Скопировать `src/OrbitWatcher/appsettings.Development.example.json` в `src/OrbitWatcher/appsettings.Development.json` и при необходимости скорректировать `Celestrack`.
2. Запустить сервер:

```bash
dotnet run --project src/OrbitWatcher
```

3. В отдельном терминале запустить тестовый клиент (проект в `tests/OrbitWatcher.SignalRClient`):

```bash
dotnet run --project tests/OrbitWatcher.SignalRClient
```

Клиент подключается к `/hubs/satellites`, логирует каждый пакет `satellitePositions`, корректно пишет ошибки соединения и попытки reconnect.

## Термины

- **OMM (Orbit Mean-Elements Message)** — формат орбитальных данных.
- **NORAD Catalog ID** — уникальный ID спутника.
- **GNSS** — спутниковые навигационные системы (GPS, GLONASS, Galileo, BeiDou и т.д.).
- **Ground track** — проекция траектории на поверхность Земли.
